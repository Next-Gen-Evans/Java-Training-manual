Java Applets — Training Manual

Short training material explaining Java Applets: basics, lifecycle, sample code and practical notes.

What is a Java Applet?

A Java Applet is a small Java program that used to run in a web browser via the Java plugin. Applets extend java.applet.Applet (or nowadays, javax.swing.JApplet) and provide a graphical interface inside a browser page.

Note: Modern browsers no longer support the Java browser plugin. Applets are deprecated and largely replaced by other technologies (Java Web Start, standalone Java apps, web technologies).

Applet Lifecycle

  • init() — Initialization, called once when the applet is first loaded.
  • start() — Called after init() and each time the applet becomes active (e.g., user returns to the page).
  • paint(Graphics g) — Called whenever the applet needs to redraw its content.
  • stop() — Called when the applet is no longer visible (e.g., user navigates away).
  • destroy() — Cleanup before the applet is unloaded.

Basic Applet Example

Below is a simple applet that displays "Hello, Applet!" — legacy-style.

// HelloApplet.java
import java.applet.Applet;
import java.awt.Graphics;

public class HelloApplet extends Applet {
    public void init() {
        // initialization code
    }
    public void paint(Graphics g) {
        g.drawString("Hello, Applet!", 20, 20);
    }
}

HTML tag (legacy) to embed an applet (obsolete):

<applet code="HelloApplet.class" width="300" height="100">
  <!-- fallback content -->
</applet>

Advantages & Disadvantages

Advantages

  • Platform-independent UI using Java.
  • Easy to embed Java code in web pages (historically).

Disadvantages

  • Requires browser plugin — now unsupported by major browsers.
  • Security restrictions (sandbox) limit functionality.
  • Deprecated and discouraged for modern web development.

Transition path for legacy applets

If you maintain legacy applets, consider:

  • Converting to a standalone Java application (Swing/JavaFX).
  • Using Java Web Start (JNLP) where supported, or packaging as an executable JAR.
  • Rewriting UI using modern web technologies (HTML5/JS) and backend services.

Practical Tips

  • Keep applet code simple and UI code separate from logic.
  • Use repaint() carefully to avoid CPU-heavy loops.
  • Test in legacy environments if you must support applets (old Java Runtime + old browser).
